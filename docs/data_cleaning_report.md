# 数据清洗报告

## 1. 数据概况

本项目使用模拟电商订单数据进行练习。原始订单表共 `180` 条记录，字段包括订单ID、下单日期、客户ID、省份、城市、商品类别、商品名称、数量、单价、折扣、支付方式、订单状态和客户评价。

## 2. 处理目标

本次处理重点包括：

- 检查订单ID重复情况；
- 统一下单日期格式；
- 清理客户ID、省份、城市字段；
- 统一商品类别口径；
- 检查数量、单价、折扣字段异常；
- 统一支付方式与订单状态；
- 对客户评价文本进行基础分类标注。

## 3. 清洗规则

### 3.1 订单ID

以订单ID作为重复检查字段。若同一订单ID出现多次，不直接删除，而是输出到重复订单清单，便于后续结合商品、金额、日期等字段复核。

### 3.2 下单日期

将 Excel 日期序列号、斜杠日期、中文日期等格式统一为 `yyyy-mm-dd`。无法识别的日期进入日期异常清单。

### 3.3 客户与地区字段

对客户ID、省份、城市字段进行前后空格清理。缺失记录单独输出，暂不自动填补。

### 3.4 商品类别

将同义类别合并为标准类别，如“数码产品”“digital”“电子产品”统一为“数码”。

### 3.5 数量、单价、折扣

数量要求为正整数；单价要求为正数；折扣统一为 0 到 1 之间的小数。异常数据不直接修正，统一进入复核清单。

## 4. 输出结果

| 文件 | 内容 |
|---|---|
| `cleaned_orders.xlsx` | 清洗后完整订单数据 |
| `duplicate_orders.xlsx` | 重复订单ID清单 |
| `date_errors.xlsx` | 日期异常清单 |
| `region_errors.xlsx` | 客户与地区异常清单 |
| `category_errors.xlsx` | 商品类别异常清单 |
| `quantity_errors.xlsx` | 数量异常清单 |
| `price_discount_errors.xlsx` | 单价与折扣异常清单 |
| `payment_status_errors.xlsx` | 支付方式与订单状态异常清单 |
| `data_quality_summary.xlsx` | 数据质量汇总 |

## 5. 处理小结

本项目按照“保留原始字段、新增标准化字段、异常单独输出”的方式进行处理。这样既保留了原始数据痕迹，也能为后续统计分析、人工复核和数据库导入提供相对稳定的数据基础。
